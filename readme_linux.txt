1. 실행 환경 준비
(컴파일과 라이브러리 생성이 완료되었다는 전제 하에 수행한다)
(타겟 보드 조작을 위한 시리얼 포트는 필요에 따라 해당 타겟 보드에 연결/해제한다.)
1.1 호스트의 IP를 192.168.10.100으로 설정한다.
1.2 서버용 타겟 보드 설정
1.2.1 호스트와 서버용 타겟 보드를 이더넷 케이블로 연결한다.
1.2.2 타켓 보드의 IP를 192.168.10.111로 설정한다.
1.2.3 호스트와 타겟 보드를 nfs를 사용해 연결한다.
1.2.4 타겟 보드의 root에 Design 디렉토리의 connect4 폴더를 복사한다.
1.2.5 nfs 마운트를 해제한다.
1.2.6 dd.sh 셸 스크립트를 실행하여 push, fnd, led 디바이스 드라이버를 삽입한다.
1.2.7 node.sh 셸 스크립트를 실행하여 push, fnd, led 디바이스 노드를 생성한다.
1.3 클라이언트용 타겟 보드 설정
1.3.1 호스트와 클라이언트용 타겟 보드를 이더넷 케이블로 연결한다.
1.3.2 타켓 보드의 IP를 192.168.10.110로 설정한다.
1.3.3 호스트와 타겟 보드를 nfs를 사용해 연결한다.
1.3.4 타겟 보드의 root에 Design 디렉토리의 connect4 폴더를 복사한다.
1.3.5 nfs 마운트를 해제한다.
1.3.6 dd.sh 셸 스크립트를 실행하여 push, fnd, led 디바이스 드라이버를 삽입한다.
1.3.7 node.sh 셸 스크립트를 실행하여 push, fnd, led 디바이스 노드를 생성한다.
1.4 서버용 타겟 보드와 클라이언트용 타겟 보드를 이더넷 케이블로 연결한다.

2. 라이브러리 컴파일
2.1 connect4 폴더의 mylib 폴더로 이동한다.
2.2 make clean으로 기존 파일을 삭제한다.
2.3 make로 라이브러리 파일을 생성하고 등록한다.

3. 디바이스 드라이버 컴파일
3.1 dd 폴더로 이동한다.
3.2 push_dd, fnd_dd, led_dd 각각의 폴더에 대하여
3.3 make clean으로 기존 파일을 삭제한다.
3.4 make로 디바이스 드라이버를 생성한다.

4 4. 실행 파일 컴파일
4.1 connect4 폴더로 이동한다.
4.2 make clean으로 기존 파일을 삭제한다.
4.3 make로 실행 파일을 생성한다.

5. 게임 실행 방법
5.1 1번에 기술된 대로 실행 환경을 준비한다.
5.2 서버 프로그램 실행
5.2.1 서버용 타겟 보드에 minicom을 통해 시리얼 포트로 접속한다.
(nfs는 해제된 상황)
5.2.2 복사한 connect4 폴더의 connect4_serv 파일을 실행한다.
5.3 클라이언트 프로그램 실행
5.3.1 클라이언트용 타겟 보드에 minicom을 통해 시리얼 포트로 접속한다.
(nfs는 해제된 상황)
5.3.2 복사한 connect4 폴더의 connect4_clnt 파일을 실행한다.
